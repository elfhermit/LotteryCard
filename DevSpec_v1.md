# 樂透刮刮樂吉祥卡 (Lottery Scratch Card)

## 1. 專案描述
開發一個靜態網頁形式的「樂透刮刮樂吉祥卡」，目標部署於 GitHub Pages。內容為數位刮刮樂，使用者刮開卡片後可獲得隨機的馬年吉祥話。
考量使用者多透過行動裝置訪問，網頁必須採用 Responsive Design (RWD) 並針對手機觸控體驗最佳化。

## 2. 核心功能需求 (MVP)
1.  **互動刮刮樂**:
    *   網頁中央顯示一張刮刮樂圖案 (Canvas 實作)。
    *   使用者可透過滑鼠 (桌面) 或手指 (手機) 進行塗抹刮除動作。
    *   **UI/UX 備註**: 手機端需設定 `touch-action: none` 於 Canvas 區域，防止刮除時觸發頁面滾動。
2.  **隨機吉祥話**:
    *   刮開覆蓋層後，顯示中獎訊息或祝福語。
    *   需準備 **10 句馬年吉祥話** (例如：「馬到成功」、「龍馬精神」等)，每次重新開始時隨機選取顯示。
3.  **行動裝置適配**:
    *   RWD 設計，確保在不同尺寸的手機螢幕上皆能滿版或置中顯示，避免跑版。
    *   支援 Touch Event，確保觸控刮除的流暢度。
4.  **重置機制**:
    *   提供「再來一張」或「重刮一次」按鈕，點擊後重置刮刮卡與隨機吉祥話。
5.  **外部配置檔**:
    *   將吉祥話清單獨立於 `config.json` 或 `settings.js`，方便非工程人員未來自行擴充或修改文案。

## 3. 進階功能建議 (Feature Wishlist & Feasibility)
為提升使用者黏著度與趣味性，建議加入以下功能，並已評估技術可行性：

1.  **沉浸式體驗 (Sound & Haptics)**:
    *   **音效**: 刮除時的「沙沙」聲、揭曉結果時的慶祝音效 (如鞭炮聲、銅板聲)。
        *   *Tech Note*: 瀏覽器會阻擋自動播放 (Auto-play Policy)。音效需在使用者第一次互動 (點擊開始或第一次觸摸刮除) 後才能解鎖播放。
    *   **震動回饋**: 在手機上刮除時配合 Taptic Engine 進行微幅震動。
        *   *Tech Note*: 使用 `navigator.vibrate()` API。 **注意：iOS Safari 目前不支援此 API**，此功能將僅限 Android 裝置體驗，iOS 用戶將自動降級 (Graceful Degradation)。

2.  **視覺特效 (Visual Effects)**:
    *   **慶祝動畫**: 當刮開面積超過一定比例 (如 70%) 自動揭曉並播放五彩碎紙 (Confetti) 或煙火特效。
    *   **動態背景**: 背景可有喜氣洋洋的動態元素 (如飄落的紅包、元寶)。
    
3.  **社群分享 (Social Share)**:
    *   **圖文卡片生成**: 允許將當前的吉祥話與背景合成一張圖片。
        *   *Tech Note*: 使用 `html2canvas` 生成圖片。
        *   *UX Note*: 網頁無法直接「一鍵發布」到 Instagram Stories。建議流程為：生成圖片 -> 彈出預覽視窗 -> 引導使用者「長按儲存」或使用 `navigator.share()` (如果裝置支援檔案分享) 喚起原生分享選單。

4.  **收藏與紀錄 (Collection)**:
    *   **集郵冊**: 使用 LocalStorage 紀錄使用者刮過哪些吉祥話，集滿 10 個給予特殊獎勵 (如「截圖炫耀」按鈕)。

## 4. 遊戲彩蛋設計 (Easter Eggs)
隱藏的小驚喜，增加話題性：
1.  **「潔癖王」獎勵**:
    *   如果使用者將銀漆刮得非常乾淨 (例如 > 98% 面積)，觸發特殊對話框：「你的堅持令人敬佩！祝你今年運勢也一樣清清白白、順順利利！」
    *   *Tech Note*: 計算透明像素會消耗效能，建議在 `touchend` 或 `mouseup` 事件後才進行一次性計算，避免在刮除過程中即時計算導致卡頓。
2.  **隱藏的「金馬獎」**:
    *   設定極低機率 (例如 1%) 出現一張「黃金刮刮卡」，刮開後不是文字，而是「特大吉」的動態金馬奔騰動畫。
3.  **開發者密技 (Konami Code)**:
    *   輸入特定手勢或點擊順序 (例如點擊標題 5 次)，解鎖「必中模式」或顯示隱藏的搞笑吉祥話 (工程師的自嘲)。
    *   *UX Note*: 手機版難以輸入上下左右指令，建議簡化為「連續快速點擊標題 5 次」觸發。
4.  **除夕夜限定**:
    *   偵測使用者裝置時間，如果是除夕或初一當天，背景音樂自動更換為更熱鬧的過年歌，且刮出的卡片樣式變為豪華版。

## 5. 技術規格與部署
*   **Frontend**: HTML5, CSS3, JavaScript (Vanilla or simple Setup).
*   **Library**: 
    *   `canvas-confetti`: 輕量級慶祝特效。
    *   `howler.js`: 處理音效相容性與 Audio Context。
    *   `html2canvas`: 截圖生成功能。
*   **PWA (Progressive Web App)**: 加入 `manifest.json` 與 Service Worker，讓使用者能將網頁「加入主畫面」，體驗更接近原生 App。
*   **Hosting**: GitHub Pages.
